# ИИ-помощник учителя истории

Десктопное приложение на Python/PySide6 для генерации тестов по истории из готовых JSON-учебников. Локальная GGUF-модель выбирает подходящие задания, а сами вопросы берутся из файлов в `Materials`.

## Как скачать готовую программу без Python

1. Откройте страницу проекта на GitHub.
2. Перейдите в раздел **Releases**.
3. В блоке **Assets** скачайте архив `СКАЧАТЬ-ГОТОВОЕ-ПРИЛОЖЕНИЕ-windows.zip`.
4. Распакуйте архив в любую папку.
5. Откройте папку `ИИ-помощник учителя`.
6. Положите GGUF-модель в папку `models`.
7. Запустите файл `ИИ-помощник учителя.exe`.

Не скачивайте `Source code (zip)` и не запускайте `install.bat`, если вам нужна готовая программа. Это исходный код для разработки, там потребуется Python и библиотеки.

GGUF-модель не входит в архив, чтобы файл не весил несколько гигабайт. После первого выбора или после размещения файла в папке `models` приложение будет само загружать модель при следующем запуске.

## Куда положить модель

Самый удобный вариант для переноса на другой компьютер:

```text
ИИ-помощник учителя\
  ИИ-помощник учителя.exe
  models\
    Qwen3.5-4B.Q4_K_M.gguf
```

Также приложение проверяет папку:

```text
Документы\ИИ-помощник учителя\Модели
```

Если моделей несколько, нужный файл можно выбрать вручную в разделе настроек модели.

## Настройка для ноутбука 8 ядер / 16 ГБ ОЗУ

Для быстрого подбора тестов используйте модель 3B-4B в квантовании `Q4_K_M`.
Оптимальный вариант для скорости: `Qwen3.5-4B.Q4_K_M.gguf` или другая
инструкционная GGUF-модель 3B-4B.

Приложение само подбирает параметры под такое железо:

- контекст: около `6144` токенов, чтобы модель стабильно работала на 16 ГБ;
- потоки CPU: физические ядра процессора, обычно `8`;
- батч: обычно `768`, чтобы быстрее обрабатывался список вопросов перед выбором;
- загрузка модели: через `mmap`, без блокировки всей оперативной памяти.

Для самой быстрой работы положите модель рядом с программой:

```text
ИИ-помощник учителя\
  ИИ-помощник учителя.exe
  models\
    Qwen3.5-4B.Q4_K_M.gguf
```

После первого запуска приложение будет автоматически находить и загружать эту
модель. Если ноутбук без NVIDIA GPU или установлена CPU-сборка, программа
будет работать в стабильном CPU-режиме.

## Как собрать EXE локально

На Windows установите Python 3.10 или новее, затем запустите из корня проекта:

```bat
build_exe.bat
```

Готовое приложение появится здесь:

```text
Test_generator\dist\ИИ-помощник учителя\ИИ-помощник учителя.exe
```

## Автосборка на GitHub

В проекте есть workflow `.github/workflows/build-windows.yml`.

- Ручная сборка: GitHub → **Actions** → **Build Windows EXE** → **Run workflow**.
- Сборка релиза: создайте тег вида `v1.0.1`, и GitHub прикрепит ZIP с EXE к Release.

## Запуск из исходников

```powershell
cd Test_generator
pip install -r requirements.txt
python main.py
```

## Где хранятся пользовательские файлы

После запуска EXE настройки, история тестов, модели и экспорт Word-документов хранятся в:

```text
Документы\ИИ-помощник учителя
```
