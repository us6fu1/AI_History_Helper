# AI Helper — генератор тестов по истории

## Что это

Десктопное приложение на **Python** с интерфейсом **PySide 6**. Тесты собираются **только из готовых JSON-банков вопросов**: модель в формате **GGUF** (`llama-cpp-python`) подбирает номера заданий по теме (ответ вида `{"pick":[…]}`), тексты вопросов берутся из файлов банка. По умолчанию приложение само подбирает CPU-потоки и включает NVIDIA GPU, если установлена подходящая GPU-сборка `llama-cpp-python`.

GGUF-модель не входит в приложение. Её нужно скачать отдельно и выбрать в разделе настроек модели. Это сделано специально, чтобы будущий установщик не весил несколько гигабайт.

## Где лежит проект

Код приложения — в каталоге **`Test_generator`** (этот файл тоже здесь).

Рядом удобно держать папку **`Materials`** с JSON-банками (`Вопросы`). Пути ниже даны так, будто текущая папка уже **`Test_generator`**.

## Установка

### Зависимости

Откройте терминал в **`Test_generator`** и выполните:

```powershell
pip install -r requirements.txt
```

Подключаются пакеты:

- PySide6 — окно приложения  
- llama-cpp-python — работа с `.gguf`  
- python-docx — экспорт в Word  

Приложение **не читает PDF-учебники** и не строит RAG-индексов — зависимость PyMuPDF не используется.

### Windows

При необходимости используйте **`install.bat`** в этой же папке — он ставит зависимости и создаёт рядом папку для моделей. Для GPU-ускорения нужна сборка `llama-cpp-python` с CUDA; обычная CPU-сборка будет работать, но GPU-слои не включит.

### Linux и macOS

Скрипт **`install.sh`** — установка зависимостей; при необходимости можно выбрать сборку под CUDA или Metal для `llama-cpp-python`.

## Быстрый старт

### Модель

Скачайте файл модели `.gguf` и укажите путь к нему в приложении в разделе настроек модели.

Рекомендуемые варианты:

- `Qwen2.5-3B-Instruct-Q4_K_M.gguf` — легче для слабых компьютеров;
- `Qwen2.5-7B-Instruct-Q4_K_M.gguf` — качественнее, но требует больше памяти;
- любая другая instruct-модель в формате GGUF, если она нормально отвечает JSON.

Удобное место для хранения модели на Windows:

```text
Документы\ИИ-помощник учителя\Модели
```

Ссылки:

- https://huggingface.co/Qwen/Qwen2.5-3B-Instruct-GGUF
- https://huggingface.co/Qwen/Qwen2.5-7B-Instruct-GGUF

### Запуск интерфейса

```powershell
python main.py
```

### Добавление банка

В приложении откройте раздел **«Банки вопросов»** и укажите папку с `*.json` (в каждом файле должен быть массив `questions`).

## Структура каталогов

| Элемент | Назначение |
|---------|------------|
| `main.py` | Точка входа, запуск GUI |
| `data/` | Настройки, список банков (`textbooks.json`), история тестов, при желании кэш генерации |
| `textbook_registry.py` | Реестр папок с JSON-банками |
| `../Materials/Вопросы/…` | Готовые банки: папки с `*.json` и массивом `questions` |

## Как подбираются вопросы из банка

- По теме и подсказкам считается релевантность элементов банка (TF-IDF внутри `question_bank.py`).  
- Модель получает только короткий «список номеров» и возвращает JSON с выбранными позициями; тексты вопросов копируются из JSON банка.

## Системные требования

- Python **3.10** и новее  
- Оперативная память: от **8 ГБ**, для средних GGUF моделей желательно **16 ГБ**  
- Свободное место: модели GGUF занимают несколько гигабайт
